====ACDC all spawn decompiler для Зова Припяти ====

Для запуска программы установите ActivePerl


Использование:   acdc_cop -d *.spawn [-o outdir] [-w] [-l] [-split_spawns] [-ls] [-split_spawns] [-graph]
		 acdc_cop -c all.ltx [-f flag1[,...]] [-a level:point] [-o outfile] [-ls]

Ключи:
-d 		- декомпиляция
-с 		- компиляция
-ls 		- декомпиляция/компиляция level.spawn
-a		- поменять при компиляции позицию актора
-w		- конвертировать апдейт-пакеты некоторого оружия в формат ЗП 
(для спавнов, сделанных с помощью aiwrapper -upgrade 3502)
-l		- разбирать спавн по вертексам согласно гейм-графу ТЧ
(для спавнов, сделанных с помощью aiwrapper -upgrade 3502)

-split_spawns - разбор all.spawn на level.spawn и level.game для каждого уровня. ACDC должен лежать 
в папке gamedata/spawns, причем в папке gamedata/levels должны лежать содержащиеся в спавне уровни, или хотя бы
level.spawn от них. Скрипт сохранит старые level.spawn и level.game как *bak файлы, и запишет на их место новые.
Достоинство такого способа - сохраняются оригинальные имена граф-поинтов.

-graph - используется совместно с -split_spawns. C таким ключом скрипт прочитает граф-поинты из section4.bin.
Достоинство такого способа - нет необходимости в оригинальных level.spawn, но оригинальные имена граф-поинтов 
будут заменены на заново сгенерированные. Это происходит потому, что в section4.bin не хранятся имена граф-поинтов.
Восстановятся только имена граф-поинтов, являющихся пунктами назначения левел-чейнджеров.
